# Combat UI and UX

Use this skill for combat-facing interface work: initiative tracker, battle readability, dice display, action affordances, map overlays, turn feedback, or damage/state communication.

## Goal

Improve combat clarity without bloating the interface or masking the underlying rules state.

## Combat UX principles

- The current actor must be obvious.
- HP / conditions / action result changes must be visible quickly.
- Turn transitions must be noticeable but not theatrical.
- Dice display should support comprehension, not delay interaction.
- Tactical features must remain optional unless the project explicitly becomes tactics-first.

## Evaluate every proposed change against

1. Readability under pressure
2. Mobile tolerance
3. Accessibility impact
4. Implementation complexity
5. Dependence on real rules/state support
